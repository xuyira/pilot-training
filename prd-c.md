# 模块 C PRD：多无人机网格规划分配训练

## 1. 模块定位

模块 C 面向飞行员/任务指挥角色的认知训练，不训练直接操纵，而训练以下能力：

- 空间规划：在网格环境中规划可行航路
- 资源匹配：将不同类型无人机分配给合适目标
- 多机并行调度：在多架无人机同时执行时持续安排后续任务
- 重规划能力：在危险区变化或高优先级目标出现后快速修正
- 路径效率控制：在可达前提下尽量接近最短可行路径
- 连续任务管理：同一架无人机可连续完成多个目标

模块 C 与前两个模块的区别：

- 模块 A：持续监控与响应控制
- 模块 B：规则驱动的任务分配
- 模块 C：空间约束下的多无人机路径规划与任务分配

## 2. 训练目标

模块 C 的训练目标不是做成战术对抗游戏，而是做成一个可量化、可比较、可调难度的网格规划任务。系统重点训练：

- 在障碍和禁飞限制下保持稳定规划
- 在多目标与多机型混合条件下做正确匹配
- 在执行中继续安排后续目标，减少空闲
- 在局面变化时进行有效重规划
- 在时间压力下兼顾正确性与路径最优性

## 3. 核心玩法概述

屏幕中心显示二维网格地图。地图上分布：

- 红色格：危险区/禁飞区，不可通过
- 若干目标点：按颜色与图标区分类型
- 我方无人机：按机型与颜色区分能力

玩家需要在限定时间内，为多架无人机规划路径并指派目标，使其完成尽可能多且尽可能正确的任务。

核心闭环：

1. 玩家左键选中一架无人机
2. 玩家右键连续点击网格，添加该无人机未来的路径点
3. 玩家按 `Enter`，该无人机开始沿当前已规划路径执行
4. 无人机执行过程中，玩家可以继续操作其他无人机
5. 无人机执行过程中，也可以继续为该机追加未来路径点
6. 无人机到达目标点后完成任务，并停留在当前位置，等待下一次调度

## 4. 地图与对象定义

### 4.1 网格地图

地图采用规则正方形网格。每个格子状态如下：

- `empty`：可通行空白格
- `danger`：危险格，不可进入
- `spawn`：我方无人机初始部署格
- `target`：任务目标格

第一版采用四方向移动：

- 上
- 下
- 左
- 右

不支持斜向穿越。这样便于：

- 统一最短路径计算
- 简化判分
- 降低玩家理解成本

视觉上无人机在相邻格中心之间作直线动画，因此“看起来是直线移动”，但规则仍然按格点路径结算。

### 4.2 目标点类型

第一版建议实现以下 4 类目标：

- 绿色 `recon`：侦查区，只能由侦察型无人机完成
- 蓝色 `intercept`：敌机/敌目标，只能由歼击型无人机完成
- 黄色 `relay`：通信中继点，只能由中继型无人机完成
- 橙色 `supply`：补给/投送点，只能由运输型无人机完成

为后续版本预留但第一版不实现的目标类型：

- 紫色 `jam`：电子压制/干扰源
- 青色 `escort`：护航接应点
- 白色 `verify`：确认识别点
- 金色 `priority`：极高优先级目标
- 灰色 `decoy`：诱饵/干扰目标

### 4.3 无人机类型

第一版建议实现以下 4 类无人机：

- 绿色 `recon_drone`：侦察机，只能完成 `recon`
- 蓝色 `fighter_drone`：歼击机，只能完成 `intercept`
- 黄色 `relay_drone`：中继机，只能完成 `relay`
- 橙色 `cargo_drone`：运输机，只能完成 `supply`

为避免玩法过度单一，后续可扩展“多能型”无人机，但第一版默认每架无人机只擅长一种目标类型。

## 5. 交互规则

### 5.1 基本操作

- 左键：选中无人机
- 右键：在网格上追加路径点
- `Enter`：让当前选中的无人机执行当前已规划路径
- `Space`：暂停/继续当前选中无人机
- `Backspace`：删除最后一个尚未执行的路径点
- `C`：清空当前选中无人机尚未执行的未来路径
- `Esc`：提前结束 block

路径相关操作规则：

- `右键`、`Enter`、`Space`、`Backspace`、`C` 只作用于当前选中的无人机
- `Esc` 为全局操作，不要求先选中无人机
- 如果当前没有选中无人机，路径相关操作不生效

### 5.1.1 选中提示

为避免多机并行时玩家混淆，当前选中无人机必须有明确提示：

- 无人机本体外圈高亮
- 当前无人机已规划路径高亮显示
- 右侧状态栏显示当前选中无人机编号、类型、状态与剩余路径长度
- 若无人机处于暂停状态，需额外显示“已暂停”

### 5.2 路径执行规则

模块 C 采用“混合式路径执行”：

- 玩家可以只规划 1 个路径点后按 `Enter`
- 玩家也可以一次规划多个路径点后按 `Enter`
- `Enter` 的语义始终为：执行“当前已经画出的整段路径”

这意味着：

- 不要求必须一次规划到最终目标
- 允许分段执行
- 允许整段执行
- 高水平玩家可通过提前规划降低空闲与操作碎片化

### 5.3 多机并行规则

- `Enter` 只作用于当前选中的一架无人机
- 一架无人机开始移动后，不会阻止玩家操作其他无人机
- 多架无人机可以同时处于飞行中
- 玩家可以在 A 机飞行时切换去给 B 机规划与下令

### 5.4 飞行中追加路径与重规划

第一版允许飞行中继续规划未来路径，也允许由玩家主动暂停后重规划：

- 若无人机正在执行，玩家仍可选中它
- 新的右键路径点会追加到当前未执行完的路径末尾
- 已经执行过的路径不可修改
- 尚未执行的未来路径可以继续追加
- 尚未执行的未来路径可以通过 `C` 清空后重画
- 玩家可按 `Space` 暂停当前无人机，再清空未来路径并重规划

第一版不实现：

- 拖拽中间路径节点改线
- 修改已经执行过的轨迹

### 5.5 完成任务后的连续调度

- 无人机到达目标格即视为完成该任务
- 完成后无人机停在当前格
- 该格成为其下一任务的起点
- 同一架无人机允许连续完成多个目标
- 玩家可在其完成上一个任务后继续为其规划后续任务

## 6. 运动与时间规则

### 6.1 速度定义

第一版不采用真实物理速度，统一采用“格速”：

- 速度单位：格/秒
- 无人机在相邻格之间匀速移动
- 所有无人机默认使用相同格速
- 难度提升优先通过目标数量、障碍数量和时间压力实现，不优先通过机型差速实现

界面可显示：

- 当前速度
- 当前剩余路径长度
- 预计到达时间

### 6.2 默认速度参数

第一版默认速度可设为：

- `L1-L2`：2.0 格/秒
- `L3`：2.2 格/秒
- `L4`：2.4 格/秒
- `L5`：2.6 格/秒

速度上升不是为了增加操作难度本身，而是为了提升并行调度与重规划压力。

### 6.3 路径生成与风险结算

玩家右键指定的是路径关键点。若新关键点与当前路径末端不相邻，系统自动按当前地图状态补齐两点之间的局部最短段。这样可以降低操作负担，同时保留“由玩家决定去哪里”的训练重点。

第一版默认：

- 玩家指定的是路径关键点
- 系统在关键点之间自动补齐局部最短可行段
- 系统不会对潜在危险路线做教学式预警
- 若补齐后的真实执行路线在之后进入危险格，则按坠毁处理

执行时风险结算规则：

- 玩家规划路径时，不提供“将进入危险区”的主动提醒
- 只要无人机实际进入危险格，立即判定坠毁
- 坠毁无人机立刻从当前局面中移除
- 该无人机尚未完成的后续路径队列全部失效
- 系统记录一次严重错误并扣分

此设计的目的不是帮助玩家避错，而是训练其自行评估路径风险。

## 7. 任务生成与局面结构

模块 C 不采用手工固定题，而采用“地图模板 + 参数化生成”的方式构建训练局面。

每个局面包含：

- 地图尺寸
- 危险区密度
- 无人机数量
- 各机型组合
- 目标数量
- 目标类型组合
- 目标优先级
- 是否出现动态危险区
- 是否出现新任务插入

生成原则：

1. 根据等级确定地图尺寸与对象数量范围
2. 放置危险区并保证关键区域不完全封死
3. 生成无人机与目标点
4. 校验至少存在可行正确解
5. 计算每个目标的理论最短可行路径长度
6. 记录推荐分配与参考总成本

生成器只保证“局面存在可行正确解”，不保证“玩家不会犯错”。因此允许以下情况在玩家操作中发生：

- 错误机型到达目标
- 低效绕路
- 错误优先级
- 飞入危险区导致坠毁

## 8. 目标与分配规则

### 8.1 硬规则

默认硬规则：

1. 目标必须由对应类型无人机完成
2. 无人机必须通过可通行路径到达目标
3. 无人机不能进入危险格
4. 任务只在无人机实际到达目标格时完成
5. 无人机一旦进入危险格立即坠毁并退出当前局面

### 8.2 软规则

用于评分但不一定阻止执行的软规则：

- 高优先级目标应优先于低优先级目标
- 总路径长度应尽量接近参考最优值
- 空闲无人机应尽量少
- 不应出现明显无效绕路
- 动态变化后应及时重规划

### 8.3 错误机型到达目标

模块 C 不禁止玩家让错误机型飞到目标点。结算规则如下：

- 错误机型到达目标点时，该目标不算完成
- 目标点继续保留在地图上
- 记录一次 `wrong_type_arrival`
- 玩家仍可调度正确机型稍后完成该目标
- 错误机型不会因此坠毁，但会计入错误分配与无效路径成本

### 8.4 连续任务链

每架无人机可形成任务链：

- `drone_1: spawn -> target_A -> target_C -> target_E`
- `drone_2: spawn -> target_B -> target_D`

系统需要记录：

- 每架无人机完成的目标序列
- 每段路径长度
- 每段完成时间
- 串行任务中的冗余移动长度

其中“串行任务中的冗余移动长度”是指：

- 一架无人机连续执行多个目标时
- 实际行驶长度超出各段理论最短可行长度的额外部分
- 用于反映连续任务链中的低效率绕路

## 9. 动态事件

第一版可支持两类动态事件：

### 9.1 危险区更新

地图中部分空白格临时变为危险区。系统只更新地图状态，不对玩家做策略提示。

玩家可自行决定是否：

- 选中相关无人机
- 按 `Space` 暂停
- 清空尚未执行的未来路径
- 重新规划

若玩家未及时处理，导致无人机继续进入新危险区，则按坠毁处理。

### 9.2 新目标插入

block 进行中临时新增高优先级目标，要求玩家：

- 评估是否打断当前分配
- 选择合适无人机插入新任务
- 平衡在途任务与新任务的优先级

## 10. 判分结果类型

每次目标结束或路径事件结束时，系统可记录以下结果类型：

- `task_completed_correct`
- `wrong_type_arrival`
- `danger_violation`
- `drone_crashed`
- `path_blocked`
- `suboptimal_path`
- `late_completion`
- `priority_error`
- `replan_success`
- `replan_failure`

## 11. 核心指标

模块 C 建议输出以下训练指标。

### 11.1 正确性指标

- `assignment_accuracy`
  - 正确类型无人机完成的目标占比
- `task_completion_rate`
  - 已完成目标数 / 总目标数
- `priority_completion_rate`
  - 高优先级目标完成率
- `danger_violation_count`
  - 进入危险格次数
- `drone_loss_count`
  - 坠毁无人机数量

### 11.2 路径质量指标

- `path_optimality_ratio`
  - 实际路径长度 / 理论最短可行路径长度
- `extra_distance`
  - 实际长度减去理论最短长度后的冗余步数
- `invalid_path_attempts`
  - 试图追加不可行路径的次数
- `serial_task_extra_distance`
  - 连续任务链中的冗余移动长度

### 11.3 调度效率指标

- `mean_planning_time_ms`
  - 从任务出现到首次下达可执行路径的平均时间
- `drone_idle_time_ratio`
  - 所有无人机空闲时间占比
- `queue_extension_count`
  - 飞行中追加路径的次数
- `multi_task_chain_mean`
  - 每架无人机平均连续完成目标数

### 11.4 重规划指标

- `replan_response_time_ms`
  - 动态事件后首次有效重规划的时间
- `replan_success_rate`
  - 动态事件后成功恢复可行路径的比例

### 11.5 综合指标

- `planning_score`
  - 综合正确性、效率、最优性与重规划表现

建议第一版综合评分示意：

- 正确完成任务：50%
- 路径最优性：20%
- 优先级控制：15%
- 重规划成功：10%
- 低空闲与低碎片化操作：5%

坠毁惩罚建议额外单独计入总分扣减项：

- 每坠毁 1 架无人机，额外扣除固定分值
- 若某类唯一可用机型坠毁导致后续目标无法完成，则这些目标按未完成结算

## 12. 难度等级设计

模块 C 采用固定 5 级难度。每级通过以下维度共同提升：

- 地图尺寸
- 危险区密度
- 无人机数量
- 目标数量
- 目标类型数
- 高优先级目标比例
- 动态事件数量
- 时间压力

### 12.1 L1

定位：单机基础路径规划

参数建议：

- 地图尺寸：`8 x 8`
- 危险区比例：`8%`
- 无人机数量：`1`
- 目标数量：`2`
- 目标类型：`1` 种，仅 `recon`
- 高优先级目标：`0`
- 动态危险区：`0`
- 新目标插入：`0`
- 每类机型冗余数量：不涉及
- 飞行速度：`2.0 格/秒`
- block 时长：`120 s`
- 单目标理论最短路径长度范围：`4-10`

训练重点：

- 理解网格移动
- 理解选机与画路径
- 理解 `Enter` 执行路径
- 建立基础避障概念

### 12.2 L2

定位：双机基础分配与串行任务

参数建议：

- 地图尺寸：`10 x 10`
- 危险区比例：`12%`
- 无人机数量：`2`
- 目标数量：`4`
- 目标类型：`2` 种，`recon + intercept`
- 高优先级目标：`1`
- 动态危险区：`0`
- 新目标插入：`0`
- 机型冗余：至少 1 类目标存在唯一正确机型
- 飞行速度：`2.0 格/秒`
- block 时长：`150 s`
- 单目标理论最短路径长度范围：`5-14`

训练重点：

- 在两架无人机间做正确机型匹配
- 开始形成“一机完成后接下一机”的连续任务意识
- 学习减少无人机空闲

### 12.3 L3

定位：多类型目标与明显资源错配

参数建议：

- 地图尺寸：`12 x 12`
- 危险区比例：`16%`
- 无人机数量：`3`
- 目标数量：`6`
- 目标类型：`3` 种，`recon + intercept + relay`
- 高优先级目标：`2`
- 动态危险区：`1` 次/每 block
- 新目标插入：`0-1` 次/每 block
- 机型冗余：部分目标类型只有 1 架对应无人机
- 飞行速度：`2.2 格/秒`
- block 时长：`180 s`
- 单目标理论最短路径长度范围：`6-18`

训练重点：

- 在资源有限下做更优分配
- 开始处理“飞行中追加路径”
- 学习第一次重规划

### 12.4 L4

定位：高并行压力下的滚动重规划

参数建议：

- 地图尺寸：`14 x 14`
- 危险区比例：`20%`
- 无人机数量：`4`
- 目标数量：`8`
- 目标类型：`4` 种，`recon + intercept + relay + supply`
- 高优先级目标：`2-3`
- 动态危险区：`2` 次/每 block
- 新目标插入：`1-2` 次/每 block
- 机型冗余：至少 1 类高优先级目标依赖稀缺机型
- 飞行速度：`2.4 格/秒`
- block 时长：`210 s`
- 单目标理论最短路径长度范围：`8-22`

训练重点：

- 多架无人机同时执行时持续调度
- 动态封锁后快速清理未来路径并重画
- 兼顾优先级与总路径效率

### 12.5 L5

定位：高密度目标、高动态变化、高时间压力

参数建议：

- 地图尺寸：`16 x 16`
- 危险区比例：`24%`
- 无人机数量：`5`
- 目标数量：`10-12`
- 目标类型：`4` 种
- 高优先级目标：`3-4`
- 动态危险区：`3` 次/每 block
- 新目标插入：`2-3` 次/每 block
- 机型冗余：存在“关键机型稀缺”局面，坠毁后可能出现部分目标无法完成
- 飞行速度：`2.6 格/秒`
- block 时长：`240 s`
- 单目标理论最短路径长度范围：`10-28`

训练重点：

- 保持多机持续运转
- 在高扰动环境中维持正确分配
- 尽量压低冗余路径和反复操作
- 在高优先级目标插入时快速重排

## 13. 难度梯度背后的设计逻辑

模块 C 的难度提升不是简单增加障碍，而是叠加以下负荷：

- 从单机到多机
- 从单类型目标到多类型目标
- 从静态地图到动态危险区
- 从一次一任务到连续任务链
- 从可从容规划到必须边执行边追加与重排

因此，高等级训练的不是手速本身，而是：

- 计划维持
- 注意切换
- 重规划速度
- 多对象并行控制

## 13.1 关于无人机是否全部显示

第一版默认：当前局面中所有可用无人机都显示在主地图上。

原因：

- 玩家需要直接看到全局资源位置
- 模块 C 的核心之一是空间调度，而不是列表式分配
- 无人机位置本身就是后续目标选择的重要依据

显示规则：

- 所有未坠毁无人机都显示在地图上
- 正在执行的无人机显示实时位置与路径
- 已暂停无人机显示暂停标识
- 已坠毁无人机从主地图移除，不再可选

## 13.2 关于坠毁后任务如何处理

若某架无人机坠毁：

- 该无人机立即从当前局面移除
- 该无人机未来路径队列清空
- 若地图上仍存在可替代的正确机型，则玩家可用其他无人机完成该目标
- 若该目标对应机型已全部损失，且不存在替代机型，则该目标将无法完成
- block 结束时，该目标按“未完成目标”结算

这类“因资源损失导致不可完成”的情况应被视为玩家决策后果的一部分，而不是系统错误。

## 14. 页面布局定义

页面采用“左主右辅”的固定布局。

### 14.1 左侧主区域

左侧约占页面宽度的 `2/3`，用于显示训练主场景：

- 网格地图
- 危险区
- 目标点
- 所有当前可用无人机
- 无人机路径
- 无人机实时移动

主区域必须优先保证：

- 地图可读性
- 路径区分度
- 选中状态明显
- 多架无人机同时运动时不混乱

### 14.2 右侧上方提示区

右侧上方为状态与提示区，用于显示“当前局面信息”，而不是教学式引导。

建议显示内容：

- 当前 block 信息：模块名、等级、剩余时间
- 当前得分概览：已完成目标数、未完成目标数、坠毁数
- 当前选中无人机信息：
  - 编号
  - 类型
  - 状态：待命 / 飞行中 / 已暂停 / 已坠毁
  - 当前位置
  - 剩余路径长度
  - 预计到达时间
- 当前局面事件信息：
  - 新目标出现
  - 危险区更新
  - 高优先级目标数量

右上提示区不应显示：

- “请暂停并重规划”
- “该路径危险”
- “建议选择哪架无人机”
- 任何直接替玩家做决策的引导

允许显示的提示风格应为事实型，例如：

- `新高优先级目标出现：T07`
- `区域 C-4 至 D-5 已转为危险区`
- `无人机 F2 已坠毁`
- `当前选中：R1（侦察机）`

### 14.3 右侧下方规则区

右侧下方用于显示固定规则说明，便于玩家在训练中随时查看。

建议内容：

- 操作说明
- 目标颜色与机型对应关系
- 坠毁规则
- 错误机型到达目标的结算规则
- 评分重点说明

此区域应为静态规则区，不随着局面变化频繁刷新。

## 15. 第一版实现边界

第一版只实现：

- 5 级固定难度
- 网格地图与危险区
- 4 类目标点
- 4 类单能力无人机
- 左键选机、右键画关键点、`Enter` 执行
- 飞行中追加未来路径
- 完成后停留原地并继续接任务
- 动态危险区更新
- 新目标插入
- 路径最优性与调度效率指标

第一版暂不实现：

- 斜向移动
- 敌机主动移动
- 战斗动画与武器判定
- 燃油/弹药/高度等复杂平台参数
- 两机协同同时到点约束
- 路径节点拖拽编辑
- 真实连续物理寻径
- 自适应难度

## 16. 推荐实现结构

### 15.1 前端状态

前端需要维护：

- 地图格子状态
- 无人机当前位置
- 无人机当前执行路径
- 无人机未来排队路径
- 当前选中无人机
- 目标池状态
- 动态事件状态
- 统计指标缓存
- 坠毁无人机列表

### 16.2 关键日志事件

建议记录：

- `drone_selected`
- `path_point_added`
- `path_execute`
- `path_append_during_flight`
- `drone_paused`
- `drone_resumed`
- `path_cleared`
- `target_completed`
- `wrong_type_arrival`
- `danger_zone_blocked`
- `danger_zone_entered`
- `drone_crashed`
- `dynamic_hazard_spawned`
- `new_target_spawned`
- `replan_submitted`
- `block_completed`

### 16.3 任务生成约束补充

任务生成器需要额外满足以下约束：

- 初始局面下，每个目标必须至少存在 1 架正确机型可达
- 高优先级目标必须保证可解
- 动态危险区事件不能把所有正确解同时封死
- 新目标插入时必须保证至少存在 1 条正确解
- 可有意制造“机型稀缺”与“坠毁后部分目标不可完成”的高压局面，但不能让局面在初始生成时无解

### 16.4 结果页建议显示

- 完成目标数
- 高优先级目标完成数
- 任务分配正确率
- 平均路径最优比
- 危险区违规次数
- 平均重规划响应时间
- 各无人机连续任务链长度
- 综合规划得分

## 17. 一句话定义

模块 C 是一个“在网格障碍环境下，对多架不同能力无人机进行持续路径规划、目标分配与动态重规划”的认知训练模块。
